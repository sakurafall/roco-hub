# 01 · MVP-A 的 P0 / P1 / P2 硬拆分 + 验证门槛 + 止损策略

> 本文件回答三个问题：**MVP-A 到底做什么；做到什么标准算赢；不赢了怎么办。**

---

## 一、MVP-A 的总目标

- **时间窗口**：Phase 2（3 周）+ Phase 3（1 周）+ Phase 4（1 周）= 5 周
- **交付平台**：微信小程序（主战场）+ H5（辅助传播）
- **用户承诺**：**查精灵 / 查克制 / 算孵蛋，一个工具搞定，速度比竞品快，观感比竞品好。**
- **不承诺**：社区、攻略生产、博主生态、登录注册流程、个人中心复杂化

---

## 二、P0 / P1 / P2 分级原则

| 级别 | 含义 | 上线态度 | 允许的简化程度 |
|---|---|---|---|
| **P0** | 首发必须稳定，体验必须打磨 | 所有用户默认看到 | 不允许简化；任何残缺都延期 |
| **P1** | 首发可带，允许简化 | 所有用户默认看到 | 允许功能裁剪（如孵蛋只支持首批精灵） |
| **P2** | 可灰度开关、可延后 | 默认关闭，后端开关开启 | 允许只上 30% 用户、只上特定场景 |

**硬规则：** P0 任何一项不达交付标准，整个 MVP-A 不上线。P1 可以延期到 MVP-B。P2 可以直接砍。

## 三、P0 清单（首发必须稳定）

### P0-1 · 精灵图鉴列表

- **范围**：
  - 卡片网格（三列）展示首批 150-200 只精灵
  - 顶部搜索框 + 属性快捷筛选（5+更多）+ 下拉筛选（形态/蛋组/数值排序/赛季）
  - 吸顶分类、滚动到底提示、骨架屏
- **交付标准**：
  - 首屏渲染 ≤ 800ms（中端机，Wi-Fi）
  - 支持下拉刷新 + 触底加载（哪怕数据是本地缓存）
  - 属性筛选可多选，有"已筛选条件"可视化标签
  - 空态有引导文案

### P0-2 · 精灵详情页

- **范围**：
  - 顶部精灵立绘 + 属性 + 编号 + 收藏按钮 + 分享按钮
  - 种族值（6 维雷达图）+ 身高/体重/蛋组
  - 简介文案
  - 克制关系简览（"弱于 X / 强于 Y"）
  - 技能列表（只读，数据来自首批整理）
  - 进化链（若首批数据已覆盖，否则隐藏模块）
  - 底部常驻"分享 / 收藏 / 进工具"按钮组
- **交付标准**：
  - 详情数据走本地缓存优先，1s 内渲染完成
  - 雷达图使用 Canvas（非 echarts 重库）
  - 所有信息必须带"数据来源 + 更新时间"小标签

### P0-3 · 搜索

- **范围**：
  - 名称搜索（支持拼音前缀，如 "dm" → 迪莫）
  - 编号搜索（3 位数字，如 "001"）
  - 搜索历史（本地存储，最多 20 条，可清除）
  - 无结果态的引导文案（"换个关键词试试 / 查看全部"）
- **交付标准**：
  - 本地首批数据走内存索引，输入后 ≤ 100ms 出结果
  - 支持模糊匹配（编辑距离 ≤ 2）
  - 历史记录可单条删除 + 一键清空
- **不做（留给 P1/P2）**：
  - 热门搜索榜（需要后端统计）
  - AI 兜底搜索（延后到三期）

### P0-4 · 收藏

- **范围**：
  - 精灵详情页收藏按钮，点击切换收藏态，带动画反馈
  - "我的"页面 → "我的收藏"列表
  - 收藏态本地存储 + 调用后端接口双写
- **交付标准**：
  - 无网/弱网下仍可完成收藏（本地先记录，网络恢复补传）
  - 收藏动画使用震动反馈（`uni.vibrateShort`）
  - 支持列表批量取消收藏
- **不做**：
  - 收藏夹分组（留给一期）
  - 收藏更新提醒（留给三期）

### P0-5 · 静默登录

- **范围**：
  - 小程序端：首次进入调用 `wx.login` 获取 code，交换后端 `openid` → 签发 JWT
  - H5 端：生成 `deviceId`（UUID，localStorage 持久化）→ 换取 JWT
  - Token 本地存储，启动时自动刷新
  - 不弹任何授权弹窗（不 `getUserProfile`、不要昵称头像）
- **交付标准**：
  - 首次进入无任何授权打扰
  - Token 失效自动续签
  - 后端必须支持账号迁移（H5 的 deviceId 在小程序端登录后可合并）→ 见 `06-api-contract-and-tech-stack.md`

### P0-6 · 主题 A（洛克日）

- **范围**：
  - CSS 变量驱动的完整 Token 系统（颜色/圆角/阴影/字号）
  - 18 属性色谱（见 `00-charter`）完整落地
  - 暗色模式切换（跟随系统 + 手动覆盖）
  - "我的 → 外观"页面可切换
- **交付标准**：
  - 切换主题无需刷新
  - 所有核心页面（首页/列表/详情/工具）在默认主题下 100% 可用
  - 暗色模式对比度符合 WCAG AA 级

### P0-7 · 本地缓存 + 数据版本号

- **范围**：
  - 精灵/技能/道具静态数据通过 CDN JSON 下发
  - 启动时比对本地版本号 + CDN 版本号，增量拉取
  - 强制缓存策略：数据有则优先本地；后台刷新
- **交付标准**：
  - 离线可用（无网状态下图鉴仍能完整查看）
  - 缓存大小上限 5MB，超出 LRU 淘汰
  - 数据版本号符合 `YYYY.MM.PATCH` 规范（见 `02-data-model`）

### P0-8 · 基础埋点

- **范围**：
  - 关键事件 20 个（详见 `04-competitor-edge-and-metrics.md`）
  - 事件批量上报（5 秒或 20 条触发一次）
  - 离线事件队列（最多 200 条）
- **交付标准**：
  - 上报失败自动重试（指数退避）
  - 后端有落库 + 基础看板能看到"今日 DAU / 搜索次数 / 收藏数"

## 四、P1 清单（首发可带，允许简化）

### P1-1 · 属性克制（单次查询版）

- **范围**：
  - 一个独立工具页：左侧选"攻击属性"，右侧选"防御属性"，输出倍率与建议文案
  - 属性列表 18 项，图标化展示
  - 支持反查（某属性被哪些克制）
- **简化形式**：
  - 不做 18×18 完整矩阵热力图（留给 MVP-B）
  - 不做"阵容克制覆盖分析"（留给一期）

### P1-2 · 孵蛋推测 V1

- **范围**：
  - 输入：蛋尺寸（m）、蛋重量（kg）
  - 输出：候选精灵列表（最多 6 只，按相似度排序）
  - 每张候选卡可跳转精灵详情
- **简化形式**：
  - 不做历史记录（留给 MVP-B）
  - 不做"仅显示同乘精灵"开关（留给 MVP-B）
  - 不做反查精灵父母（留给一期）
  - **仅支持首批 150-200 只精灵**，页面明确提示"当前支持精灵范围"
- **算法**：
  - 简单欧氏距离：`sqrt((inputSize - spirit.size)^2 + (inputWeight - spirit.weight)^2)`
  - 距离阈值 + 排序返回

### P1-3 · 分享卡 V1

- **范围**：
  - 精灵详情页"分享"按钮 → 生成精美海报
  - 海报包含：立绘、名称、编号、属性、种族值、产品 Logo + 二维码
  - 小程序走 `onShareAppMessage` + `onShareTimeline`
  - H5 走 Canvas 绘制 → `saveImageToPhotosAlbum`
- **交付标准**：
  - 生成时间 ≤ 1.5s
  - 图片大小 ≤ 300KB
  - 扫码/点击分享图能回到对应精灵详情页（带 utm 埋点参数）
- **铁律复述**：脱离产品本体也有内容价值，进入产品后又有进一步价值。

### P1-4 · 主题 B（深海光年）

- **范围**：
  - 第二套完整 Token 覆盖 + 玻璃拟态背景
  - 可在"我的 → 外观"切换
- **简化形式**：
  - 初版可以仅提供纯色背景，不做背景图集（留给 MVP-B）
  - Lottie 动画延后

## 五、P2 清单（可灰度开关、可延后）

### P2-1 · 详情页增强动效

- 雷达图数值切换动画
- 克制关系呼吸灯
- 页面进出转场

### P2-2 · 分享卡第二模板

- 克制结果卡（攻 × 防 + 倍率 + 建议文案）
- 孵蛋结果卡（候选 Top 3 + 输入数据）

### P2-3 · 数据可信标签完整展示

- 每条数据末尾展示"来源 / 更新时间 / 可信度徽章"
- 长按可查看完整溯源

### P2-4 · 首次进入 60 秒引导

- 品牌 Logo 启动动画（500ms）
- 功能一句话介绍 + Skip 按钮
- "首次收藏"引导气泡
- "首次搜索"引导气泡

## 六、P0/P1/P2 的灰度策略

- **P0** 全员默认开启。
- **P1** 全员默认开启，但保留后端"一键降级"开关（出线上问题可快速关闭此功能而不影响 P0）。
- **P2** 默认关闭，后端配置"灰度名单 / 灰度百分比"开启。
- 灰度通道：后端下发 `features` 字段到前端，前端按 `featureFlags[name]` 判断是否渲染。

```json
{
  "features": {
    "share_card_v1": true,
    "theme_b": true,
    "hatch_predict_v1": true,
    "detail_animation": false,
    "onboarding_60s": false
  }
}
```

## 七、验证门槛（Phase 4 末做第一次检查）

### 7.1 七个关键指标

| # | 指标 | MVP-A 关卡目标 | 测量方式 |
|---|---|---|---|
| 1 | DAU | ≥ 500 | 后端 `user_active_daily` 聚合 |
| 2 | 次日留存 | ≥ 30% | 首启次日再次打开比例 |
| 3 | 7 日留存 | ≥ 12% | 首启后 7 天内任一天再次打开 |
| 4 | 每 DAU 搜索次数 | ≥ 3 | `event.search_submit` / DAU |
| 5 | 人均精灵详情浏览数 | ≥ 4 | `event.spirit_detail_view` / DAU |
| 6 | 收藏率 | ≥ 20% | 当日收藏 UV / DAU |
| 7 | 分享触发率 | ≥ 5% | `event.share_card_generate` / DAU |
| 8（扩展） | 孵蛋功能使用率 | ≥ 10% | `event.hatch_predict_submit` UV / DAU |

### 7.2 指标采集与看板

- 所有指标的原始事件定义见 `04-competitor-edge-and-metrics.md`
- Phase 2 末必须上线简单看板（哪怕只是后端跑定时任务 + 飞书机器人日推）
- Phase 4 末必须能看到最近 14 天完整数据

### 7.3 关卡判定规则

| 情况 | 判定 | 行动 |
|---|---|---|
| 7 个核心指标全部达标 | **绿灯** | 进入 P5（MVP-B），加码投入 |
| 5-6 项达标，关键项（1/4/7）达标 | **黄灯** | 做针对性优化 2 周，再检查 |
| ≤ 4 项达标，或关键项不达标 | **红灯** | 进入止损流程 |

## 八、止损策略（Fail-Fast）

### 8.1 核心原则

> **失败时不是终止项目，而是快速退回到更窄的问题定义。**

### 8.2 两周微检查点（Phase 2 中途）

在 P2 第 2 周结束时做一次轻量检查（仅看开发进度 + 核心链路可用性），不看数据。

- 如果 P0 主链路（图鉴列表 → 详情 → 搜索 → 收藏）不通，**立即停止新功能开发，全员攻坚**。
- 如果 API 契约有分歧/频繁变更，**暂停前端开发，先锁契约**。

### 8.3 四周关卡检查点（Phase 4 末）

按第 7 章判定规则执行。**红灯时的收敛路径：**

#### 路径 A · 退守"孵蛋 + 图鉴"双拳头

- 砍掉：克制工具（大部分用户不用）、分享卡
- 集中：把孵蛋做成"行业唯一真正可用的计算器"
- 品牌口径：改为"洛克孵蛋精灵助手"
- 适用条件：指标 8 达标但其他不达标

#### 路径 B · 退守"只做微信小程序"

- 砍掉：H5 侧所有运维
- 集中：优化小程序体验，投放微信生态（搜一搜、服务号）
- 适用条件：H5 DAU 极低，小程序指标还行

#### 路径 C · 退守"只做热门精灵工具"

- 砍掉：图鉴全量（数据维护成本高）
- 集中：50 只热门精灵的深度数据（完整技能、蛋组、努力值、推荐流派）
- 品牌口径："洛克热门精灵精研助手"
- 适用条件：用户只看少量精灵，长尾精灵没人查

#### 路径 D · 暂停并复盘

- 停止新功能开发 2 周
- 做 30 个真实用户访谈
- 重新定义产品
- 适用条件：A/B/C 都不清晰

### 8.4 止损决策责任人

- 提议者：任何团队成员
- 决策者：产品负责人（你）
- 输入：14 天指标数据 + 用户访谈摘要 + 团队共识
- 时间窗：判定红灯后 3 个工作日内必须决策

---

## 九、出局条件（Kill Switch）

以下任一情况，立即暂停项目并重新评估：

1. 游戏官方直接发函或公开表示不欢迎
2. 小程序被微信下架且申诉未通过
3. 数据源全部失效且无替代
4. 核心开发人员离开 ≥ 50%
5. 连续 8 周 WAU 低于 300
